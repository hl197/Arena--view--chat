# PRD | ArenaView 前端 UI 优化 · 手绘手账风

> 版本：v1.0（最终版）
> 日期：2026-06-28
> 项目类型：前端产品 / UI 重构
> 视觉风格：手绘手账风

---

## 1. 目标与背景

### 1.1 项目背景

当前 ArenaView 共有 5 个页面（首页、辩论页、结果页、历史页、设置页），辩论页为微信风格全屏布局，历史记录和结果都需要跳转独立页面查看，交互链路长。整体 UI 偏朴素，缺乏产品辨识度和记忆点。用户输入问题后直接跳转到讨论页等待，缺少"创建群聊"的仪式感。

### 1.2 优化目标

将核心讨论页升级为 **手绘手账风格 + 两栏式布局**（左侧历史 + 右侧群聊），增加群聊创建仪式感动效，强化用户主持人身份。首页保持独立（欢迎+输入），设置页保留。四条硬指标：

1. **视觉辨识度**：全套手绘手账风统一，一眼能认出产品调性
2. **交互流畅度**：查看历史无需跳转页面，操作路径减少 1 步
3. **仪式感提升**：群聊创建过程有涟漪扩散入场动效，用户感受到"一群 AI 来帮我分析"
4. **页面精简**：从 5 个页面精简为 3 个（首页 / 讨论页 / 设置页）

### 1.3 成功指标

- 历史记录使用率提升 50%+（侧边栏常驻 vs 顶部导航点击）
- 用户首次使用视觉记忆度提升
- 群聊创建环节用户流失率下降
- 页面数量减少，信息架构更清晰

### 1.4 不做什么

- 不改变后端逻辑和 SSE 事件协议
- 不新增 Agent 类型或辩论机制
- 不做移动端适配（本次只优化桌面端）
- 不做快捷键支持

## 2. 用户与场景

### 2.1 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---------|------|---------|
| **决策困境者** | 面临重大选择（买房/跳槽/创业），需要多角度分析 | 快速获得全面视角、决策不后悔 |
| **内容创作者** | 写文章/做方案，需要 AI 帮着想角度 | 高效 brainstorm、灵感激发 |
| **终身学习者** | 对复杂话题好奇，想深入理解 | 沉浸式学习、多角度看问题 |

### 2.2 典型使用场景

**场景 1：首次访问 → 发起讨论**
> 作为新用户，我打开首页看到欢迎信息和输入框，输入我的问题后，看到一群 AI 专家"加入群聊"，然后开始讨论。

**场景 2：讨论中查看历史**
> 作为老用户，我在讨论一个问题时，想随时翻看之前的讨论记录，不用离开当前讨论。

**场景 3：查看结论**
> 作为用户，辩论结束后我想在同一个界面里一边看讨论过程，一边对照决策地图。

**场景 4：配置 API Key**
> 作为用户，我需要一个独立的页面来配置我的 API Key 和其他设置。

## 4. 功能详情

### 4.1 页面架构调整

**调整前（5 个页面）**：
```
/ → 首页（深色风格，输入问题）
/debate/:sessionId → 辩论页（微信风格，全屏）
/result/:sessionId → 结果页（深色风格，决策地图）
/history → 历史页（深色风格，列表）
/settings → 设置页（深色风格，API Key）
```

**调整后（3 个页面）**：
```
/ → 首页（手绘风，欢迎 + 输入 + 示例 + 工作原理）
/debate/:sessionId → 讨论页（手绘风，两栏布局）
/settings → 设置页（手绘风，API Key + 偏好设置）
```

**页面流转**：
```
首页 → 输入问题提交 → 跳转 /debate/:sessionId → 显示涟漪入场动效 → 进入群聊
讨论页 → 点击左侧历史卡片 → 切换到对应讨论（URL 同步更新）
讨论页 → 顶部 Logo → 返回首页
讨论页/首页 → 顶部设置 → 跳设置页
```

**顶部导航**：
- 左侧：ArenaView Logo（点击回首页）
- 右侧：设置按钮
- 移除"历史"入口（已合并到讨论页左侧）

### 4.2 两栏式讨论页布局

**交互说明**：
- 点击左侧历史卡片 → 右侧切换到对应讨论，URL 更新为 `/debate/:sessionId`
- Hover 历史卡片 → 右上角出现删除图标 🗑️，点击弹出确认
- 拖拽中间分割线 → 左右宽度实时变化（范围 240px - 480px，默认 300px）
- 点击折叠按钮 → 左侧收起为 60px 窄栏，只显示头像缩略图和新增按钮

**边界情况**：
- 无历史记录：左侧显示空状态手绘插画 + "开始你的第一次讨论吧"
- 历史很多：左侧滚动，搜索框和新建按钮 fixed 在顶部
- 窗口太窄（< 900px）：自动折叠侧边栏

### 4.3 涟漪扩散入场动效

**触发时机**：用户在首页提交问题后跳转到讨论页，开始播放动效。

**动效分 3 阶段**：

**阶段 1：中心出现（约 0.8s）**
- 用户头像（带皇冠 👑）从页面中心放大出现，带弹跳效果
- 下方手写体文字："正在为你组建专家团..."
- 背景是淡淡的手账本纹理

**阶段 2：涟漪扩散（约 3-4s，根据 Agent 数量）**
- 第一个 Agent 头像从中心"荡漾"出去，落到圆周第一个位置
- 伴随涟漪波纹效果（box-shadow 扩散模拟水波）
- 头像下方滑入名字标签 + 一句话身份介绍
- 每隔 0.6-0.8s 出现下一个，共 5 个（或配置的数量）
- 右下角进度文案："已邀请 2/5 位专家..."
- 出现顺序带轻微随机感，不机械

**阶段 3：群聊展开（约 0.8s）**
- 全部到位后，短暂停顿 0.3s
- 整体缩小 + 上移，头像移动到顶部成员栏位置
- 群聊界面从底部滑入展开
- 弹出一条系统消息（便签风格）："🎉 专家团已就位，开始讨论你的问题吧！"
- 随后正式进入讨论流程，SSE 连接开始接收消息

**技术实现**：
- 纯 CSS animation + transform，不引入动画库
- 头像位置用绝对定位 + 三角函数计算圆周分布
- 涟漪效果用 `::before` 伪元素 + `box-shadow` 扩散
- 动效总时长控制在 5-6 秒

### 4.4 成员面板 & 决策地图面板

**位置**：讨论页右侧，可滑出滑入，默认收起

**两个 Tab**：
- 👥 成员
- 🗺️ 决策地图

**成员面板**（宽度 280px）：
- 顶部：群聊名称 + 成员数量
- 主持人分区（用户）：头像👑 + 名字 + "主持人"标签
- 讨论成员分区（Agent）：逐个列出，头像 + 名字 + 身份一句话 + 状态徽章
- 状态徽章：思考中 🤔 / 搜索中 🔍 / 发言中 ✏️ / 已完成 ✅
- 点击某个成员 → 高亮该成员的最近一条消息

**决策地图面板**：
- 辩论结束后自动展开并切换到此 Tab
- Markdown 渲染：标题、列表、加粗、表格
- 顶部操作栏：复制全文按钮 + 折叠按钮
- 内容可滚动
- 折叠后：只显示顶部窄条（"决策地图" + 展开按钮）

## 5. 非功能需求

### 5.1 性能
- 首页首屏加载 < 2s（本地开发环境）
- 涟漪动效帧率 ≥ 60fps（不卡顿）
- 消息列表 100 条以上滚动流畅
- SVG filter 只应用于边框和装饰元素，不影响大面积区域性能

### 5.2 兼容性
- 支持 Chrome / Edge / Firefox / Safari 最新两个版本
- 最低桌面分辨率：1280 × 800
- SVG filter 不支持的浏览器自动降级为实线边框

### 5.3 可维护性
- 手绘风格统一通过 CSS 变量 + 全局 SVG filter 实现
- 基础组件封装在 `components/ui/` 下，统一管理
- design token 沉淀在 TailwindCSS 配置中，不散落

### 5.4 可访问性
- 正文文字对比度 ≥ 4.5:1
- 所有可交互元素有 `aria-label`
- 支持键盘 Tab 导航主要操作
- 动效遵循 `prefers-reduced-motion` 系统设置

## 7. 设计规范

### 7.1 视觉风格：手绘手账风

**风格定位**：手绘手账风 — 温暖、有创造力、像一本摊开的手账本

**三个核心识别特征**：
1. **手绘抖动线** — 所有边框都不是直线，带轻微抖动（SVG filter 实现）
2. **便利贴气泡** — 消息像贴在本子上的便利贴，微微倾斜
3. **纸胶带装饰** — 分割线、重要标记用彩色纸胶带效果

| 纸胶带蓝 | 蓝色 | `#a8d4f0` |

### 7.3 手绘效果实现

- 全局定义一个 SVG filter `#hand-drawn`（feTurbulence + feDisplacementMap）
- 线条粗细：2px（主边框）、1px（次级分割）
- 抖动幅度：1.2-1.5px（有手绘感但不影响阅读）
- 所有边框、按钮轮廓、卡片边框应用 filter
- **正文文字不应用 filter**（保证可读性）
- 标题、装饰文字可应用

### 7.4 组件风格

| 组件 | 风格描述 |
|------|---------|
| 按钮 | 手绘不规则边框 + 马克笔填充色 + hover 轻微按压效果 |
| 输入框 | 手绘边框 + 纸白背景 + focus 时边框变粗 + 颜色加深 |
| 卡片 | 白色/淡黄背景 + 手绘边框 + 微旋转（±0.5°~1°）+ 柔和阴影 |
| 消息气泡 | 便利贴造型 + 手绘边框 + 轻微倾斜 + 顶部小胶带装饰 |
| 分割线 | 手绘波浪线 + 两侧渐淡 / 纸胶带效果 |
| 标签/徽章 | 手绘圆角矩形 + 马克笔色填充 + 手写体字 |

### 7.5 便利贴气泡细节

- 每个气泡微微倾斜（用户消息右偏 1°，Agent 消息左偏 1°）
- 顶部有一小条"纸胶带"装饰（颜色随机从马克笔色里选）
- 阴影柔和，像贴在纸上的感觉
- 不同 Agent 的胶带颜色不同，辅助区分发言人
- 用户气泡：淡黄色 + 金色描边 + 光晕

### 7.6 字体

- 标题：手写体（中文用 "Ma Shan Zheng"，英文用 "Caveat"）
- 正文：系统字体栈（保证可读性）
- 从 Google Fonts CDN 加载手写体，提供降级 fallback

### 7.7 动效规范

- 基础动画时长：200ms（快速反馈）
- 入场动画：300-500ms
- 弹性曲线：cubic-bezier(0.34, 1.56, 0.64, 1)（spring 感）
- 缓出曲线：cubic-bezier(0.4, 0, 0.2, 1)
- 所有动效尊重 `prefers-reduced-motion` 设置
